# Broken AI: Dead-Code Protocol

## Event Overview

In the year 2032, Aetheria Systems suffers a catastrophic logic-bomb across its flagship AI stack. Models are unstable, chatbots hallucinate, and critical systems are leaking data.

Your mission is to step into the wreckage, identify corrupted logic, restore stability, and bring the system back under control before the global grid collapses.

This event is designed to test debugging skill, system thinking, code quality, and problem-solving under pressure.

## Mission Tracks

Choose a sector to start working on finish off all the bugs to save Aetheria Systems:

### Sector A: Predictive Core

The machine learning pipeline is unstable and the predictions are unreliable.

Your goal is to:

- Fix broken data and model pipelines
- Restore mathematical sanity
- Improve prediction reliability
- Optimize performance where possible

### Sector B: Neural Voice

The generative AI chatbot is broken, inconsistent, and logically unsound.

Your goal is to:

- Fix prompt logic and reasoning flow
- Improve API behavior and response quality
- Reduce hallucinations
- Restore stable, useful outputs

## Event Phases

### Phase 1: Operation Reconstruction

Duration: 150 minutes

This is the main challenge phase. Teams will:

- Analyze broken code
- Detect and fix logic bombs
- Improve the implementation
- Optimize the system for stability and quality

This phase determines the base score.

### Phase 2: Zero-Day Protocol

Duration: 30 minutes, optional

After Phase 1, teams vote on whether to activate the attack phase.

- Majority Yes: Attack phase begins
- Majority No: Phase-1 extended for 30 minutes

If activated, teams may review other repositories and report additional bugs with proof.

Each valid bug found earns a bonus point.

### Phase 3: Final Handover

Duration: 10 minutes

Submit the required deliverables based on whether the attack phase was activated.

## Submission Requirements

### If the attack phase is skipped

Submit:

- The repository with fixes applied
- A README describing the bugs fixed and improvements made

### If the attack phase is activated

Submit:

- The repository with fixes applied
- A debug report covering your fixes
- An attack report containing verified bugs found in other teams' repos

## Evaluation Criteria

Final score is based on:

- System stability
- Code quality
- Attack bonus, if applicable

## What We Expect

Strong submissions should demonstrate:

- Correctness of fixes
- Clear reasoning behind changes
- Better stability and reliability
- Clean, maintainable code
- Focused optimization without unnecessary complexity

## Team Strategy

Teams are encouraged to:

- Read the codebase carefully before modifying it
- Fix root causes rather than symptoms
- Validate changes with tests or reproducible checks
- Keep the solution stable under edge cases

## Theme

This event follows a high-stakes recovery narrative:

> Fix a broken AI system. Optimize it. Decide whether to play safe or take the attack phase to gain extra points.

## Final Goal

Restore order, protect the system, and earn the title of Ultimate Logic Architect.
